## 1. 🎯 Goal

Add **role-based access control** so RAKIT can gate content operations by:
- user role (`admin`, `editor`, `viewer`)
- action (`read`, `create`, `update`, `delete`)
- collection (`posts`, `products`, `*` wildcard)

This keeps the engine usable for real CMS projects without hardcoding authorization logic.

## 2. 🚫 Non-goals

- Row-level / attribute-based access control
- Complex policy language (Casbin/Oso)
- Multi-tenant orgs/workspaces (future)
- Audit trail UI (future)

## 3. 🌐 API Contract

### Protected content actions
RBAC is enforced automatically on content routes:
- `GET /api/v1/:collection` → `read`
- `GET /api/v1/:collection/:id` → `read`
- `POST /api/v1/:collection` → `create`
- `PUT /api/v1/:collection/:id` → `update`
- `PATCH /api/v1/:collection/:id` → `update`
- `DELETE /api/v1/:collection/:id` → `delete`

### Admin endpoints
- `GET /api/v1/admin/roles` — list roles
- `GET /api/v1/admin/permissions` — list permissions
- `POST /api/v1/admin/users/:id/role` — assign role to user

### Default roles
- `admin` → full access
- `editor` → read/create/update on permitted collections
- `viewer` → read-only
- `public` → anonymous read-only for allowed collections

## 4. 🗄️ Data Model

### Tables

```sql
roles(id, name unique, created_at, updated_at)
permissions(id, action, collection, created_at, updated_at)
role_permissions(role_id, permission_id)
user_roles(user_id, role_id)
```

### Seeded data
- roles: `admin`, `editor`, `viewer`, `public`
- permissions: combinations of action × collection (`*` wildcard allowed)
- mapping:
  - admin → all permissions
  - editor → read/create/update on `*`
  - viewer → read on `*`
  - public → read on `*`

## 7. 🧪 Tests

### Matrix
- admin can read/create/update/delete any collection
- editor can read/create/update allowed collections
- viewer can read only
- public can read only if collection has public read permission

### Manual checks
- unauthenticated `GET /api/v1/posts` → 200 if public permission exists
- unauthenticated `POST /api/v1/posts` → 401/403
- editor token `DELETE /api/v1/posts/:id` → 403
- admin token `DELETE /api/v1/posts/:id` → 200

## 8. 🔒 Security Considerations

- **Deny by default**: if no permission match, return 403
- **Fail closed**: invalid role lookup = unauthorized
- **No wildcard escalation** beyond explicit `*` seeds
- **Admin endpoints** must be protected by admin role
- Consider audit logging on future permission changes

## 9. ⚡ Performance Considerations

- Permission check should be a single DB query per request
- Cache could be added later in-memory or Redis
- Keep wildcard lookup simple (`collection = $1 OR collection = '*'`)
